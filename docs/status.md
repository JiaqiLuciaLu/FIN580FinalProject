## Phase D (full grid) — summary

Grid: λ₀ = `np.arange(0, 0.95, 0.05)` (19 values), λ₂ = `10 ** -np.arange(5, 8.25, 0.25)` (13 values).
All 3 CV folds + full fit → **988 grid CSVs**. Matches `main.R`.

`pick_best_lambda` uses `full_cv=False` to match R's `main.R:119` default (cv_3 only for valid SR).

### Validation

- **988 grid CSVs**: median diff ~1e-11 (machine precision). ~7 outlier cells at λ₀∈{0, ≥0.75} drive the tail (max SR diff 0.29, max beta diff 1.06).
  - Root cause: sklearn `lars_path` vs R `lars` package differ in how they log intra-K drop-steps (coefficient crossing zero without changing support size). Irreducible without reimplementing R's `lars`.
- **SR_N.csv**: max |Δ| = 3.83e-3 (from the same outlier cells). Curves visually identical to R.
- **Best (i*, j*)**: Python and R both pick (λ₀=0.75, λ₂=1e-8) with ΔSR = 7e-13.
- **Sanity single-cell (0.15, 1e-8)**: Selected_Ports byte-identical, Weights max |Δ| 2.28e-11, TimeSeriesAlpha max |Δ| 5.39e-11.

### Caveat on R's ground-truth files

R's `Selected_Ports_10.csv`, `Selected_Ports_Weights_10.csv`, `train/valid/test_SR_10.csv`, and `SDFTests/TimeSeriesAlpha.csv` in the shared data dir were generated by `main_simplified.R` at the single cell (0.15, 1e-8) and were **NOT regenerated** after `main.R`'s full-grid run. The 988 grid CSVs and `SR_N.csv` ARE from the full grid.

Phase D comparison logic only diffs apples-to-apples files; stale ground-truth files are diffed against the sanity-check rerun at (0.15, 1e-8) instead.

### Key implementation notes

- `metrics._load_sr_cache` / `_pick_best_from_cache`: one-shot CSV cache avoids 46× redundant reads in `pick_sr_n` (10+ min → 11s).
- `pruning.ap_pruning(n_workers=...)`: dispatches via `multiprocessing.Pool.imap_unordered` with fork-based shared memory (module-level `_WORKER_CTX`).
- `metrics.pick_sr_n` writes `SR_N.csv` with a hand-rolled writer to match R's `write.table` quoted-header format (`"srN","","",…`).
- Grid wall time: ~22 min on 8 cores. Per-config LARS-EN slow cells (small λ₀) dominate; don't expect 8× speedup over serial.
